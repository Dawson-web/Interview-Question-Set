# 面经数据处理与归档技能

## 概述

本技能用于处理面试经验（面经）数据，将从牛客网等平台导出的面经 Markdown 文件进行解析、去重、分类，并按"岗位/公司"的文件夹结构组织归档输出，确保数据分类清晰且易于检索。

## 工作流程

### 步骤 1：分析输入数据

接收到面经数据文件后，先确认数据格式：

- 检查是否为牛客网导出的标准 Markdown 格式
- 识别文件头部元信息（导出时间、关键词、结果数等）
- 确认每条面经的结构（标题、来源类型、链接、作者、正文）

如格式不符合预期，参考 `references/data_format.md` 了解标准格式定义，必要时对数据进行预处理。

### 步骤 2：执行解析脚本

使用核心解析脚本处理面经数据：

```bash
python3 scripts/parse_interview.py <输入文件路径> --output <输出目录> --position <岗位名称>
```

**参数说明：**

| 参数 | 必需 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `input_file` | 是 | - | 牛客网导出的 Markdown 文件路径 |
| `--output, -o` | 否 | `./interview-archive` | 输出目录路径 |
| `--position, -p` | 否 | `前端` | 默认岗位名称 |

脚本执行以下处理：

1. **解析**：从 Markdown 中提取每条面经的元数据和正文
2. **公司识别**：通过公司别名表和智能匹配识别面试公司
3. **部门识别**：针对大公司（如腾讯）识别具体部门
4. **无效过滤**：过滤纯提问帖、纯链接转载帖、内容过短帖
5. **去重**：基于链接、内容哈希和相似度三重去重
6. **分类归档**：按"岗位/公司"结构输出为分类文件

### 步骤 3：AI 结构化提取面试内容

脚本执行完成后，**由你（AI 助手）逐个读取输出目录中的面经文件**（排除 `_index.md` 和 `_summary.md`），对每个文件中每条面经的「面试内容」部分进行结构化提取和重写。

**处理目标：** 将原始的混合文本（叙述 + 问题 + 多轮混杂）转化为结构清晰的格式。

#### 提取规则

1. **叙述性内容 → 引用块（blockquote）**
   - 面试时间、地点、感受、结果、背景描述、薪资待遇等非问题内容
   - 使用 `> ` 前缀，放在面经内容最前面
   - 如无叙述性内容则省略此部分

2. **面试问题 → 扁平列表**
   - 所有面试问题提取为 `- ` 无序列表项
   - 每个问题应简洁明了，去除序号、冗余修饰
   - 如果问题包含追问（如"具体怎么实现的？"），将追问合并为同一条目或拆分为独立条目
   - **不需要按知识领域分类**，直接以扁平列表展示

3. **多轮面试 → 轮次分段**
   - 如果面经包含多轮面试（一面/二面/三面/HR面等），使用 `**一面**`、`**二面**` 等加粗标题分段
   - 每个轮次下独立列出该轮的问题列表
   - 如果只有单轮面试，不添加轮次标题

4. **特殊处理**
   - 笔试题/手撕代码题照常提取，标注为 `- 手撕：xxx` 或 `- 笔试：xxx`
   - 面试官反馈、offer 信息等归入叙述性内容（引用块）
   - 如果原始内容已经是比较清晰的问题列表格式，保持原样仅做格式统一

#### 输出格式示例

**单轮面试：**
```markdown
> 2026年2月腾讯 CSIG 前端一面，视频面试，约40分钟，已过。

- 自我介绍
- 闭包是什么？应用场景有哪些？
- Promise 的状态有哪些？Promise.all 和 Promise.race 的区别
- Vue 的响应式原理，Vue2 和 Vue3 有什么区别
- 跨域问题怎么解决
- 手撕：实现一个防抖函数
```

**多轮面试：**
```markdown
> 腾讯 IEG 前端秋招，三轮技术面 + HR面，整体体验不错，已OC。

**一面**
- 自我介绍
- CSS 盒模型，box-sizing 的作用
- 事件循环机制，宏任务和微任务的区别
- 手撕：数组去重

**二面**
- 项目中遇到的最大挑战
- Webpack 打包优化做过哪些
- HTTP 缓存策略，强缓存和协商缓存
- 手撕：实现 Promise.all

**三面**
- 职业规划
- 对前端技术发展的看法
- 反问环节
```

#### 执行方式

1. 使用 `list_dir` 获取输出目录下所有面经文件路径
2. 逐个 `read_file` 读取文件内容
3. 识别每条面经的内容区域（`### {序号}. {标题}` 到下一个 `---` 之间）
4. 对面试内容部分执行上述提取规则
5. 使用 `replace_in_file` 将提取后的结构化内容写回文件
6. 处理完所有文件后，向用户报告提取统计

### 步骤 4：验证输出结构

脚本执行并完成 AI 结构化提取后，验证输出目录结构是否正确：

```
interview-archive/
├── _summary.md              # 总汇索引：包含处理统计和分类概览
├── 前端/
│   ├── _index.md            # 岗位级索引
│   ├── 腾讯/
│   │   ├── _index.md        # 公司级索引（含部门分组）
│   │   ├── 腾讯-wxg.md      # 微信事业群面经
│   │   ├── 腾讯-csig.md     # 云与智慧产业面经
│   │   └── 腾讯-通用.md     # 未识别部门的面经
│   ├── 快手/
│   │   ├── _index.md
│   │   └── 快手.md
│   └── 其他/
│       ├── _index.md
│       └── 其他.md
```

### 步骤 5：生成统计报告

脚本会在终端输出统计报告，包含：

- 原始条目数 → 有效面经数 → 去重后条目数
- 按岗位和公司分类的统计数据
- 过滤和去重的详细数量

## 自定义扩展

### 添加新公司

在 `scripts/parse_interview.py` 的 `COMPANY_ALIASES` 字典中添加新公司及其别名：

```python
COMPANY_ALIASES = {
    '新公司名': ['别名1', '别名2', 'alias_en'],
    # ...
}
```

### 添加大公司部门细分

在 `TENCENT_DEPARTMENTS`（或新建类似字典）中添加部门关键词映射，然后在 `extract_company_info` 函数中添加对应的部门识别逻辑。

### 调整去重策略

在 `deduplicate_entries` 函数中修改：

- `similarity > 0.8`：调整相似度阈值，值越小越严格
- 扩展去重维度（如标题相似度比较）

### 调整无效过滤规则

在 `validate_entry` 函数中修改：

- `len(content) < 30`：调整最小内容长度阈值
- `ask_patterns` 列表：添加或修改提问帖的匹配模式
