# 面经数据格式参考

## 牛客网导出 Markdown 格式

### 文件头部元信息

```markdown
# 牛客抓取导出

- 导出时间：2026/2/28 11:05:06
- 关键词：腾讯前端面经
- 抓取页数：20 页
- 结果数：379 条
- 去重：是
```

### 单条面经结构

每条面经由 `---` 分隔符分隔，结构如下：

```markdown
## {序号}. {标题}
- 来源类型：{类型编号}
- 链接：{URL}
- 作者：{作者名}

{正文内容}

---
```

### 正文内容常见模式

1. **结构化面试问题**：编号列表形式，如 `1. 自我介绍 2. 闭包是什么...`
2. **面试时间线记录**：包含 📍面试公司、🕐面试时间、💻面试岗位 等 emoji 标记
3. **多轮面试合集**：使用 `一面`、`二面`、`三面` 等分段
4. **纯提问帖**：只是询问是否有面经，无实际面试内容（属无效条目）
5. **链接转载帖**：只包含 "牛客网 牛客网在线编程..." 等网站名称

### 公司和部门识别规则

#### 公司识别
- 从标题优先提取
- 匹配公司别名表（如"鹅厂"→"腾讯"）
- 从正文前 500 字符中提取

## 去重策略

1. **链接去重**：相同 URL 的条目只保留第一个
2. **内容哈希去重**：正文内容 MD5 哈希完全相同的条目去重
3. **相似度去重**：同一作者且内容前 200 字符相似度 > 80% 的条目去重

## 无效条目过滤规则

以下类型的条目将被标记为无效并过滤：
- 内容少于 30 个字符
- 纯提问帖（包含"有没有面经"、"求面经"等模式）
- 纯链接转载帖（内容只有网站名称）

## AI 结构化提取输出格式

脚本生成基础归档文件后，由 AI 对每条面经的「面试内容」部分进行结构化提取。提取后的格式如下：

### 叙述性内容（引用块）

面试时间、地点、感受、结果、背景描述等非问题内容，使用 `>` 引用块：

```markdown
> 2026年2月腾讯 CSIG 前端一面，视频面试，约40分钟，已过。
```

### 面试问题（扁平列表）

所有面试问题以 `- ` 无序列表展示，不按知识领域分类：

```markdown
- 自我介绍
- 闭包是什么？应用场景有哪些？
- Promise 的状态有哪些？
- 手撕：实现一个防抖函数
```

### 多轮面试（轮次分段）

包含多轮面试时，使用加粗标题分段：

```markdown
**一面**
- 问题1
- 问题2

**二面**
- 问题3
- 问题4
```

### 完整示例

```markdown
### 1. 腾讯 CSIG 前端面经

- 来源：牛客面经专区
- 作者：xxx
- 链接：https://...

> 腾讯 CSIG 秋招前端，两轮技术面，已OC。

**一面**
- 自我介绍
- CSS 盒模型，box-sizing 的作用

- 手撕：数组去重

**二面**
- 项目介绍
- Webpack 打包优化
- HTTP 缓存策略
- 手撕：实现 Promise.all

---
```
